Tên ca sử dụng:Make Order Mã ca sử dụng:1 Độ quan trọng:Cao
Primary Actor:Online User Use case type:Chi tiết, quan trọng
Stakeholders and Interests:
User có quyền đặt hàng, thay đổi đơn hàng.
Admin tiếp nhận đơn hàng và thay đổi đơn hàng theo yêu cầu của User
Brief Description:Use case mô tả việc khách hàng đặt hàng online
Trigger:Người dùng (đã xác thực) thăm trang web, tiến hành mua hàng
Type:External
Relationships:
  • Associations:User
  • Include:Authenticate User, Cancel Order
  • Extend:
  • Generalization:
Normal Flow of Events:
  1. User thăm trang web, thêm những CD ưa thích vào giỏ hàng
  2. User tiến hành đặt hàng (yêu cầu người dùng đăng nhập hệ thống - Authenticate User use case)
  3. User điền thông tin cá nhân
  4. User chọn các thông tin bổ sung (bắt buộc)
  5. Hệ thống sẽ tạo một hóa đơn tương ứng với đơn đặt hàng
  6. Kết thúc việc đặt hàng
Subflows:
4a: Thông tin bổ sung
  1. Cách thức vận chuyển
  2. Cách thức thanh toán
Alternate/exceptional Flows:
2a: Người dùng yêu cầu hệ thống cho lập tài khoản mới
6a: Người dùng có thể hủy đơn hàng trong thời gian cho phép
